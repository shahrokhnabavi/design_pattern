Decorator

Problem

You want to add behavior or state to individual objects at run-time. Inheritance is not feasible because it is static and applies to an entire class.
A decorator is likely to add functionality (decorate) an object and a strategy is likely to swap functionality.

Intent

  • Attach additional responsibilities to an object dynamically. Decorators provide a flexible alternative to subclassing for extending functionality.
  • Client-specified embellishment of a core object by recursively wrapping it.
  • Wrapping a gift, putting it in a box, and wrapping the box.
  • You would use it when you want the capabilities of inheritance with subclasses, but you need to add functionality at run time

Category: Structural Patterns
