Factory

Problem

Instantiating an instance of a class from a given family of classes on the fly

Intent

  • The factory class prevents “coupling” of the client code that uses a class, to the class itself.
  • It also encapsulates the construction of an object.
  • It also lets you easily return newer versions of an object to the client.
  • You can “mock” your factory in unit tests to return mock objects.

Category: Creational Patterns
